Project's API

Overview

You can currently access two endpoints: Station or Trip.
For each of these endpoints, you can specify 'info' parameter: "real-time" (default), "schedule" or "prediction".

  • STATION: query a station, on given day (today if not specified) to get information on trips.
    EXAMPLE: "{% url 'api_station' %}?uic_code=8727622&day=20170216"
    To access it: {% url 'api_station' %}
    With parameters:
    • Required: uic_code: in a 7 or 8 digits format, for instance, uic_code=8727622
    • Optional: day: in a yyyymmdd format, for instance, day=20170217. If not specified, today.
    • Optional: info: default "real-time", this parameter can be "real-time", "schedule" or "prediction".

  • TRIP: query a trip_id to get information about its stops:
    EXAMPLE: {% url 'api_trip' %}?trip_id=DUASN156466F01002-1_402666"
    To access it: {% url 'api_trip' %}
    With parameters:
    • Required: trip_id: for instance, trip_id=DUASN156466F01002-1_402666
    • Optional: info: default "real-time", this parameter can be "real-time", "schedule", "prediction".
